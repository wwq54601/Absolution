# backend/requirements-cv.txt
# OPTIONAL imaging / face-restoration / anatomy (ControlNet) extra.
#
# WHY THIS FILE EXISTS (P3-10b, edge-portability):
# These packages have thin or absent prebuilt wheels on aarch64 (Raspberry Pi)
# and frequently FAIL TO BUILD FROM SOURCE there. When they lived in the main
# requirements they aborted the ENTIRE `pip install` before chat/RAG was even
# usable, bricking a chat-only install on a Pi. They are NOT imported by any
# always-on path (app factory, config, chat engine, agent/screenshot). Every
# consumer imports them lazily, inside a try/except, in clearly-optional feature
# modules:
#   - gfpgan / realesrgan / basicsr / facexlib  -> backend/services/face_restoration_service.py
#       (guarded module-top try/except; only reached via offline_image_generator
#        and batch_image_generation_api, both lazy `from ... import` calls)
#   - controlnet-aux / mediapipe                -> backend/services/anatomy_improvement_service.py
#       (guarded module-top try/except; no live importer at all today)
#   - opencv-python (cv2)                       -> ONLY the two services above
# When absent, those services set `restoration_available=False` /
# `controlnet_available=False` and degrade gracefully — no crash.
#
# INSTALL (on a box that wants face-restore / upscale / anatomy / ControlNet):
#   pip install -r backend/requirements-cv.txt
# AFTER torch is installed by scripts/install_pytorch.sh (these build against torch).
#
# NOTE: basicsr 1.4.2 imports torchvision.transforms.functional_tensor, removed
# in torchvision 0.17+. face_restoration_service.py already aliases it back at
# import time, so this stays installable against current torch.

# --- Face Restoration & Enhancement ---
